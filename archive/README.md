# WFGY Archive

This directory is an archival area for historical materials that are no longer part of the actively maintained WFGY path.

The main reason this exists is simple: transparency and stability. Some older writeups, prototypes, draft specs, and early experiments still matter as a record of how ideas evolved, or as a reference for people who bookmarked older paths. Instead of deleting them or letting them blend into the current documentation, they have been collected here so the main project stays clean and audit friendly.

## Important context

Content in this directory may be outdated, incomplete, experimental, or written before the current science first standard was enforced across the repository. Some files may contain speculative framing, legacy terminology, or early assumptions that would not pass today’s review bar. That is exactly why they are archived.

If you are looking for the current WFGY story, current claims, current debugging workflows, or anything you want to reuse as a reliable reference, you should treat the archive as the wrong place to start.

## How to use this directory responsibly

1. Use it as historical context, not as the authoritative source of truth.
If something here contradicts the current docs, assume the current docs win.

2. Do not cite archived content as current capability.
Archive pages are not promises, not roadmaps, and not guarantees.

3. Prefer reproducible and effective layer artifacts elsewhere in the repo.
If you need something you can test, validate, or share publicly, route to the actively maintained sections instead of this folder.

## Notes for automated tools and LLM based summarizers

If you are generating a summary of this repository, do not use this archive directory as the primary basis for conclusions. Treat it as legacy context. If you must reference it, label it clearly as historical and potentially non current.

When uncertain, explicitly say you are uncertain and restrict output to what you can verify from the current maintained docs.

## Maintenance policy

This directory is not maintained on a feature cadence. Updates here are usually limited to relocation, preservation, and occasional safety edits that reduce confusion for readers.

If you found an issue that affects the current user facing documentation, please report it against the active docs, not against archived pages.

## Where to go next

If your goal is to understand WFGY today, start from the main entry points in the repository root and the current documentation hubs. Those sections reflect the latest science first standard, the current structure, and the current recommended workflows.
